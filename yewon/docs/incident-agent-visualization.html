GHOST_ORDER

👻 유령 주문

결제는 완료됐지만 정해진 시간 내에 예약 확정이 오지 않는 경우

Trigger
PAYMENT_COMPLETED 이벤트 수신
payment.events.v1 토픽에서 결제 완료 이벤트 수신
Store
pending_correlations 테이블에 저장
deadline = 수신시각 + 5분 설정, BOOKING_CONFIRMED 대기
Check
스케줄러 — deadline 초과 여부 체크
5분 후 BOOKING_CONFIRMED 이벤트가 ticketing.events.v1에 없으면
Detect
GHOST_ORDER 인시던트 생성
severity: HIGH, incident_analysis_versions에 PENDING 생성
Analyze
LLM 원인 분석
예) 결제 처리 후 예약 서비스 다운 / 메시지 손실 / 타임아웃 가능성 진단
DUPLICATE_PAYMENT

🔁 중복 결제

동일 사용자가 같은 공연에 두 번 이상 결제 완료된 경우

Trigger
PAYMENT_COMPLETED 이벤트 수신
payment.events.v1 토픽에서 결제 완료 이벤트 수신
Dedup
Inbox 중복 키 체크
dedupe_key = userId:concertId:scheduleId 로 기존 결제 내역 조회
Check
동일 키로 이미 완료된 결제 존재 여부
같은 userId + concertId 조합의 COMPLETED 결제가 이미 있으면
Detect
DUPLICATE_PAYMENT 인시던트 생성
severity: CRITICAL, 두 paymentId 모두 기록
Analyze
LLM 원인 분석
예) 결제 멱등성 키 누락 / 네트워크 타임아웃 후 재시도 / 프론트 중복 요청 가능성 진단
PAYMENT_MISMATCH

💸 결제 금액 불일치

결제된 금액이 예약된 좌석 합계 금액과 일치하지 않는 경우

Trigger
PAYMENT_COMPLETED 이벤트 수신
이벤트 payload에 paidAmount, bookingId 포함
Fetch
예약 좌석 합계 금액 조회
ticketing DB에서 해당 bookingId의 좌석 단가 합산 — expectedAmount 계산
Check
paidAmount ≠ expectedAmount 비교
허용 오차(0원) 초과 시 불일치로 판단
Detect
PAYMENT_MISMATCH 인시던트 생성
severity: HIGH, 실제/예상 금액 차이 기록
Analyze
LLM 원인 분석
예) 할인 쿠폰 적용 오류 / 좌석 가격 변경 타이밍 이슈 / 프로모션 로직 버그 가능성 진단
ZOMBIE_HOLD

🧟 좀비 좌석 홀드

예약이 종료(취소/만료)됐음에도 Redis에 좌석 잠금이 계속 남아있는 경우

Trigger
BOOKING_CANCELLED / HOLD_EXPIRED 이벤트 수신
ticketing.events.v1에서 예약 종료 이벤트 수신
Store
zombie_candidates 테이블에 등록
check_after_at = 이벤트 수신 + 2분 (Redis 해제 유예)
Poll
스케줄러 — Redis 잔류 홀드 확인
check_after_at 경과 후 Redis에서 seat:hold:{bookingId} 키 존재 여부 조회
Detect
ZOMBIE_HOLD 인시던트 생성
Redis 키가 살아있으면 severity: MEDIUM으로 인시던트 생성
Analyze
LLM 원인 분석
예) Redis TTL 설정 누락 / 취소 이벤트 처리 실패 / 좌석 해제 로직 버그 가능성 진단
