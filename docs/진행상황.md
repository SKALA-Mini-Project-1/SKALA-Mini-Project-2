## Phase 1. DB 스키마 ✅ 완료

**완료 파일**
- `infra/postgres/init.sql` — incident 스키마 및 6개 테이블 DDL

**생성된 테이블**

| 테이블 | 용도 |
|---|---|
| `incident.incidents` | 탐지 사건 원장 |
| `incident.incident_analysis_versions` | LLM 분석 이력 (버전별) |
| `incident.incident_status_history` | 상태 변경 이력 |
| `incident.detector_inbox_events` | Kafka 중복 이벤트 제거 (Inbox 패턴) |
| `incident.detector_pending_correlations` | 유령 주문 / 미확정 결제 교차 상태 추적 |
| `incident.detector_zombie_candidates` | 좀비 예약 Redis 폴링 대상 |

**주의**: PostgreSQL 컨테이너 최초 기동 시(`postgres-data` 볼륨 초기화 상태)에만 자동 실행됨. 기존 컨테이너 재사용 시 `docker compose down -v` 후 재시작 또는 psql 수동 실행 필요.

## Phase 2. incident-detector 🔄 진행중

**완료**
- [x] Step 2-1. 프로젝트 골격 (build.gradle, Application, Dockerfile)
- [x] Step 2-2. Kafka Consumer (PaymentEventConsumer, TicketingEventConsumer)
- [x] Step 2-3. Inbox 패턴 (DetectorInboxService, DetectorInboxEvent, DetectorInboxRepository)
- [x] Step 2-4. 4가지 탐지 룰
  - [x] DuplicatePaymentRule (중복 결제)
  - [x] GhostOrderRule (유령 주문 — PendingCorrelation 기반)
  - [x] UnconfirmedPaymentRule (미확정 결제 — PendingCorrelation 기반)
  - [x] ZombieHoldRule + ZombieHoldChecker + ZombieHoldCheckScheduler (좀비 예약 — Redis 폴링)
- [x] Step 2-5. IncidentWriteService + IncidentAnalysisVersion + IncidentCreateCommand DTO
- [x] Step 2-6. application.properties
- [x] PendingCorrelationCheckScheduler (deadline 초과 correlation → incident 전환)

**컴파일 검증**: ✅ `./gradlew :incident-detector:compileJava` 통과

**생성된 주요 클래스**

| 클래스 | 역할 |
|---|---|
| `rules/DuplicatePaymentRule` | PAYMENT_CONFIRMED 2회 이상 → DUPLICATE_PAYMENT incident |
| `rules/GhostOrderRule` | PAYMENT_CONFIRMED → PendingCorrelation 등록, booking.confirmed 수신 시 해소 |
| `rules/UnconfirmedPaymentRule` | PAYMENT_PAID/WEBHOOK → PendingCorrelation 등록, PAYMENT_CONFIRMED 수신 시 해소 |
| `rules/ZombieHoldRule` | booking.canceled/expired → ZombieCandidate 등록 |
| `zombie/ZombieHoldChecker` | Redis seat:access / seat:lock:owner key 잔존 확인 |
| `zombie/ZombieHoldCheckScheduler` | 주기적 ZombieCandidate 스캔 → Redis 폴링 → incident 생성 |
| `scheduler/PendingCorrelationCheckScheduler` | deadline 초과 PendingCorrelation 스캔 → GHOST_ORDER / UNCONFIRMED_PAYMENT incident 생성 |
| `incident/IncidentWriteService` | createOrUpdate 트랜잭션, severity 에스컬레이션, 상태 이력, 분석 버전 생성 |
